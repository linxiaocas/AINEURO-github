# 贡献指南

感谢你对ClawdChip CPU项目的关注！我们欢迎各种形式的贡献。

## 如何开始

1. **Fork** 本仓库
2. **Clone** 你的Fork
3. 创建**分支** (`git checkout -b feature/amazing-feature`)
4. **提交** 更改 (`git commit -m 'Add amazing feature'`)
5. **推送** 到分支 (`git push origin feature/amazing-feature`)
6. 创建 **Pull Request**

## 贡献类型

### 🐛 Bug报告
- 使用GitHub Issues
- 描述问题、重现步骤、期望行为
- 提供环境信息

### 💡 功能建议
- 在Discussions中分享想法
- 说明使用场景和价值
- 愿意参与实现

### 📝 文档改进
- 修正拼写和语法错误
- 添加示例和教程
- 翻译文档

### 🔧 代码贡献
- 遵循现有代码风格
- 添加测试
- 更新文档

## 开发流程

1. 阅读 [架构文档](./docs/architecture/overview.md)
2. 在Issues中找到任务 或 创建新Issue
3. 在Discord中讨论设计
4. 实现并测试
5. 提交PR并等待审核

## 代码规范

- **Verilog/SystemVerilog**: 遵循Google风格指南
- **Python**: PEP 8
- **Chisel**: 官方风格指南
- **文档**: Markdown，中英双语优先

## 社区准则

- 尊重他人
- 建设性讨论
- 分享知识
- 享受过程
